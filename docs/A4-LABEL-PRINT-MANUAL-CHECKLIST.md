# בדיקת הדפסת מדבקות A4 — Checklist ידני

מסמך זה משלים את הבדיקות האוטומטיות (`delivery-labels.test.ts`).  
הדפסה פיזית / Save as PDF דורשת אימות ידני על מדפסת ומכשיר אמיתי.

## פריסה

- [ ] 3 עמודות × 6 שורות = 18 מדבקות לעמוד
- [ ] A4 Portrait בלבד
- [ ] גבולות מדבקה ברורים
- [ ] רווחים קבועים בין תאים
- [ ] מעבר עמוד (page break) אחרי כל 18 מדבקות
- [ ] מדבקה אחת → עמוד אחד
- [ ] 18 מדבקות → עמוד אחד
- [ ] 19 מדבקות → שני עמודים
- [ ] 36 מדבקות → שני עמודים
- [ ] 37 מדבקות → שלושה עמודים

## תוכן

- [ ] טקסט ארוך (שם לקוח) לא שובר את הפריסה
- [ ] כתובת ארוכה נשארת קריאה
- [ ] מוצרים רבים → מדבקות המשך (continuation), ללא חיתוך שקט של מוצר
- [ ] אין הצגת `undefined` / `null` / `NaN`
- [ ] סכום מזומן לשליח מופיע במדבקה הראשית בלבד

## פעולות מסך

- [ ] הדפס מסומנים — רק המשלוחים שנבחרו
- [ ] הדפס מוצגים — כל המשלוחים במסנן הנוכחי
- [ ] כל המסננים נשמרים במסך אחרי Preview
- [ ] בחירה (selection) אינה נשמרת בענן
- [ ] Preview אינו משנה dirty / revision
- [ ] Preview מוגן ב-Session (משתמש מחובר)
- [ ] משתמש אנונימי מופנה ל-Login
- [ ] אין מידע עסקי ב-URL
- [ ] אין Blob URL בחשיפה ללקוח
- [ ] אין שינוי סטטוס / מלאי / נתונים בעת Preview או הדפסה

## מובייל (תצוגה בלבד)

- [ ] Preview קריא ב־320px / 375px / 390px / 430px
- [ ] אין גלילה אופקית בלתי מתוכננת בתצוגה
- [ ] ההדפסה עצמה נשארת A4 אמיתי (לא layout מובייל)

## הדפסה פיזית (חובה לפני קבלה סופית בשטח)

- [ ] Save as PDF — עמודים ומידות תקינים
- [ ] הדפסה ממדפסת A4 — יישור וחיתוך תאים
- [ ] בדיקה ב־iPhone (Safari) — Preview + Save PDF
- [ ] בדיקה ב־Android (Chrome) — Preview + Save PDF
